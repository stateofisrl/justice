# ScamResolvedCSI - Django Web Application

A professional fraud recovery and cyber intelligence platform built with Django.

## Features

- **Blog System**: Rich text editor with image uploads, SEO metadata, and categories
- **Support System**: Ticket management with staff-user chat functionality
- **Admin Dashboard**: Full-featured admin interface with custom actions
- **SEO Optimized**: Meta tags, sitemaps, robots.txt
- **Responsive Design**: Mobile-first Tailwind CSS styling
- **Production Ready**: Configured for Render deployment with PostgreSQL

## Quick Start (Local Development)

1. **Clone and setup**:
   ```bash
   cd cyber_intelligence
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   pip install -r requirements.txt
   ```

2. **Environment variables**:
   ```bash
   cp .env.example .env
   # Edit .env with your settings (defaults work for local dev)
   ```

3. **Database setup**:
   ```bash
   python manage.py migrate
   python manage.py seed_blog  # Creates sample posts and admin user
   ```

4. **Run server**:
   ```bash
   python manage.py runserver
   ```

5. **Access**:
   - Site: http://127.0.0.1:8000/
   - Admin: http://127.0.0.1:8000/admin/ (admin/admin123)

## Deploy to Render (Zero Manual Work)

1. **Push to GitHub**:
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin <your-repo-url>
   git push -u origin main
   ```

2. **Deploy on Render**:
   - Go to [render.com](https://render.com)
   - Click "New" → "Blueprint"
   - Connect your GitHub repo
   - Render will automatically:
     - Create PostgreSQL database
     - Install dependencies
     - Run migrations
     - Collect static files
     - Create admin user (admin/admin123)
     - Seed blog posts
     - Deploy your app

3. **Done!** Your site is live at `https://your-app.onrender.com`

## Key Features Overview

- Users receive email notifications when staff replies
- Track ticket status through resolution

### Email Configuration

Currently set to console output for development. For production, configure SMTP:

```python
EMAIL_BACKEND = 'django.core.mail.backends.smtp.EmailBackend'
EMAIL_HOST = 'smtp.gmail.com'
EMAIL_PORT = 587
EMAIL_USE_TLS = True
EMAIL_HOST_USER = 'your-email@gmail.com'
EMAIL_HOST_PASSWORD = 'your-app-password'
DEFAULT_FROM_EMAIL = 'your-email@gmail.com'
```
